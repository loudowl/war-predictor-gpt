# Product Requirements Document (PRD)

## 1. Executive Summary
This project aims to develop a cross-platform application with a web front-end in React and a mobile front-end in React Native. The app will leverage Bellingcat's toolkit and Ukraine timemap tools, along with a local Ollama AI, to generate and categorize reports on military activities. Users will interact with dynamic maps and graphs to visualize military initiatives and predict future scenarios based on AI-generated insights, supported by open-source data.

## 2. Goals & Success Metrics

### Goals
- Provide users with real-time, categorized reports on military activities.
- Offer predictive insights into future military scenarios.
- Ensure an intuitive and engaging user interface across platforms.

### Success Metrics
- User engagement: Measured by daily active users (DAU) and session length.
- Accuracy and relevance of AI-generated reports and predictions.
- User satisfaction: Average rating of 4.5+ on app stores and positive feedback through surveys.

## 3. User Personas

### Investigative Journalists
- Needs: Access to reliable and timely data on military activities.
- Goals: Use data for reporting and analysis.

### Defense Analysts
- Needs: Comprehensive insights into military initiatives.
- Goals: Assess and predict military movements and strategies.

### Policy Makers
- Needs: Data-driven insights for decision-making.
- Goals: Understand future scenarios to inform policy decisions.

## 4. Core Features

### P0 (Highest Priority)
- **Dynamic Report Generation**: Utilize AI to generate reports on military activities.
- **Interactive Maps**: Display areas of initiative strength and weakness with historical data.
- **Future Scenario Simulation**: Allow users to simulate future scenarios with AI-driven insights.

### P1 (High Priority)
- **Data Filtering and Sorting**: Let users filter and sort reports using various categories.
- **Graphical Representation**: Visualize the direction of military initiatives over time.

### P2 (Medium Priority)
- **Cross-Verification with Open Sources**: Integrate open-source data verification for AI predictions.
- **User Feedback Mechanism**: Collect user feedback for continuous improvement.

## 5. User Stories

- **As an investigative journalist**, I want to generate detailed reports on military activities so that I can use them in my articles.
- **As a defense analyst**, I want to visualize the direction of military initiatives so that I can assess strategic advantages.
- **As a policy maker**, I want to simulate future scenarios so that I can make informed decisions.

## 6. Out of Scope

- Integration with non-Bellingcat data sources not specified in the brief.
- Advanced AI training beyond existing capabilities for long-term predictions.
- Real-time collaboration features among users.

## 7. Technical Constraints

- The app must be built using React for web and React Native for mobile.
- Integration with Bellingcat’s toolkit and Ukraine timemap is mandatory.
- Local deployment of Ollama AI for report generation.

## 8. Timeline Estimate

### Phase 1: Research & Planning (4 weeks)
- Finalize requirements and validate with stakeholders.
- Conduct technical feasibility study.

### Phase 2: Design & Prototyping (6 weeks)
- Develop wireframes and UI/UX designs.
- Create prototypes for user feedback.

### Phase 3: Development (12 weeks)
- Implement core features (P0).
- Begin integration with Bellingcat tools and AI systems.

### Phase 4: Testing & QA (4 weeks)
- Conduct user testing and iterate based on feedback.
- Perform bug fixes and refine features.

### Phase 5: Launch & Post-Launch Support (4 weeks)
- Deploy app to web and mobile platforms.
- Monitor user feedback and address critical issues.

By adhering to this structured approach, we aim to deliver a robust and user-friendly application that meets the needs of our target users and provides actionable insights into military activities.
